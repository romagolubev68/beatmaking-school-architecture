## 3.1. Тема и концепция

* **Название приложения:** BeatMaster Academy
* **Описание:** Интерактивная платформа для обучения музыкальному продюсированию и созданию битов. Пользователи могут проходить курсы по работе в DAW (FL Studio, Ableton Live), изучать теорию музыки, сведение и мастеринг, а также получать обратную связь от опытных менторов.
* **Целевая аудитория:** Начинающие и опытные битмейкеры, музыкальные продюсеры, студенты музыкальных вузов и все, кто хочет освоить создание современной музыки с нуля.
* **Ключевой функционал:**
    * Просмотр каталога курсов с фильтрацией по сложности и ПО.
    * Персонализированный личный кабинет со статистикой обучения.
    * Доступ к закрытому контенту после авторизации и оплаты.
    * Профили менторов с портфолио.

## 3.3. Технологический стек

### Frontend
* **React.js:** Библиотека для создания компонентного интерфейса. Позволяет быстро обновлять данные без перезагрузки страниц.
* **React Router:** Для организации навигации между 8 разделами приложения (SPA).
* **Redux Toolkit:** Для централизованного управления состоянием (данные пользователя, состояние корзины).

### Backend
* **Node.js & Express:** Среда выполнения и фреймворк для быстрой обработки API-запросов.
* **MySQL:** Реляционная система управления базами данных. Выбрана за высокую производительность и надежность.

### Инфраструктура
* **Vercel:** Для хостинга фронтенд-части.
* **Railway / Docker:** Для развертывания сервера и контейнеризации базы данных MySQL.

## 3.4. Визуальное проектирование интерфейса (Вариант В)


#### 1. Страница: Главная (`/`)
* **Тип доступа:** Публичный.
* **Основные блоки:** * Шапка (Логотип, ссылки на каталог и наставников, кнопка входа).
    * Hero-секция (Заголовок "Освой искусство создания битов", кнопка призыва к действию).
    * Секция «Популярные курсы» (Карточки с превью).
    * Футер (Контакты, соцсети).
* **Динамический контент:** Блок популярных курсов загружается через `GET /api/courses?popular=true`.

#### 2. Страница: Аутентификация (`/auth/*`)
* **Тип доступа:** Публичный.
* **Основные блоки:** * Переключатель Вход / Регистрация.
    * Поля ввода (Email, Password, Full Name для регистрации).
    * Ссылка «Забыли пароль?».
* **Логика:** Данные отправляются на `POST /api/auth/login` или `POST /api/auth/register`.

#### 3. Страница: Каталог курсов (`/courses`)
* **Тип доступа:** Публичный.
* **Основные блоки:** * Панель фильтров (по софту: FL Studio, Ableton; по сложности).
    * Сетка карточек курсов (Заголовок, цена, рейтинг).
* **Динамический контент:** Список меняется при выборе фильтров (`GET /api/courses?category=...`).
* **Состояния:** Скелетоны при загрузке, заглушка "Ничего не найдено" при пустом результате.

#### 4. Страница: Детальная страница курса (`/courses/:id`)
* **Тип доступа:** Публичный.
* **Основные блоки:** * Обложка курса.
    * Подробное описание и программа обучения.
    * Карточка покупки (Цена + кнопка "В корзину").
* **Динамический контент:** Все данные подтягиваются по ID из `GET /api/courses/:id`.

#### 5. Страница: Личный кабинет (`/profile`)
* **Тип доступа:** Приватный (только для авторизованных).
* **Основные блоки:** * Блок редактирования профиля (Смена аватара, имени, почты).
    * Секция безопасности (Изменение пароля).
* **Динамический контент:** Личные данные пользователя из `GET /api/user/me`.

#### 6. Страница: Мое обучение (`/dashboard`)
* **Тип доступа:** Приватный.
* **Основные блоки:** * Список купленных курсов с прогрессом.
    * Секция «Последний просмотренный урок».
* **Динамический контент:** Список формируется на основе связей в БД (`GET /api/user/my-courses`).

#### 7. Страница: Наставники (`/mentors`)
* **Тип доступа:** Публичный.
* **Основные блоки:** * Карточки преподавателей (Фото, специализация, ссылка на SoundCloud/YouTube портфолио).
* **Динамический контент:** Список всех менторов из `GET /api/mentors`.

#### 8. Страница: Оплата (`/checkout`)
* **Тип доступа:** Приватный.
* **Основные блоки:** * Список товаров к оплате.
    * Выбор способа оплаты.
    * Форма ввода данных.
* **Динамический контент:** Итоговая сумма рассчитывается на основе данных корзины.

## 3.5. Схема данных и API

### Модели данных (MySQL):
1.  **Users**: `id` (INT PK), `email` (VARCHAR), `password_hash` (VARCHAR), `full_name` (VARCHAR), `created_at` (DATETIME).

2.  **Mentors**: `id` (INT PK), `full_name` (VARCHAR), `bio` (TEXT), `specialization` (VARCHAR), `portfolio_url` (VARCHAR).

3.  **Courses**: `id` (INT PK), `title` (VARCHAR), `description` (TEXT), `price` (DECIMAL), `mentor_id` (INT FK -> Mentors.id), `image_url` (VARCHAR).

### Ключевые API эндпоинты:
* `POST /api/auth/register` — Регистрация нового аккаунта.
* `POST /api/auth/login` — Аутентификация и получение токена доступа.
* `GET /api/user/me` — Получение данных профиля авторизованного пользователя.
* `GET /api/courses` — Получение списка всех курсов (с поддержкой фильтрации).
* `GET /api/courses/:id` — Детальная информация о курсе и его программе.
* `GET /api/mentors` — Список всех наставников школы.
