# Лабораторная работа №1: Проектирование архитектуры динамического веб-приложения
**Дисциплина:** Веб-технологии / Разработка клиентских приложений

---

## 3.1. Тема и концепция

* **Название приложения:** BeatMaster Academy
* **Описание:** Интерактивная образовательная платформа для начинающих и опытных музыкальных продюсеров. Приложение позволяет изучать курсы по созданию битов, сведению и мастерингу, отслеживать свой прогресс и взаимодействовать с профессиональными наставниками.
* **Целевая аудитория:** Битмейкеры, студенты музыкальных направлений, sound-дизайнеры и хобби-музыканты, стремящиеся повысить качество своих работ.
* **Ключевой функционал:**
    * Просмотр каталога курсов с фильтрацией по сложности и ПО.
    * Персонализированный личный кабинет со статистикой обучения.
    * Доступ к закрытому контенту после авторизации и оплаты.
    * Система связи с менторами (наставниками).

## 3.3. Технологический стек

### Frontend
* **React.js:** Библиотека для создания компонентного интерфейса. Позволяет быстро обновлять данные без перезагрузки страниц.
* **React Router:** Для организации навигации между 8 разделами приложения (SPA).
* **Redux Toolkit:** Для централизованного управления состоянием (данные пользователя, состояние корзины).

### Backend
* **Node.js & Express:** Среда выполнения и фреймворк для быстрой обработки API-запросов.
* **PostgreSQL:** Реляционная база данных, обеспечивающая надежное хранение связей между пользователями и их курсами.

### Инфраструктура
* **Vercel/Netlify:** Для хостинга фронтенд-части.
* **Render/Railway:** Для развертывания серверной части и базы данных.

## 3.4. Визуальное проектирование (Вариант В)

### Страница: Главная (`/`)
* **Основные блоки:** Шапка с навигацией, Hero-блок (баннер), секция "Топ курсов" (4 карточки), футер.
* **Динамический контент:** Блок "Топ курсов" загружается через `GET /api/courses/popular`.

### Страница: Каталог (`/courses`)
* **Основные блоки:** Боковая панель фильтров, сетка карточек курсов с ценами.
* **Состояния:** * *Загрузка:* Скелетоны карточек.
    * *Ошибка:* Баннер "Ошибка сети".

### Страница: Мое обучение (`/dashboard`)
* **Основные блоки:** Список активных курсов, прогресс-бары прохождения (в %), кнопка "Продолжить".
* **Динамический контент:** Данные подгружаются из `GET /api/user/courses`.

*(Аналогично продуманы остальные 5 страниц: профиль, оплата, наставники, детали курса и авторизация).*

## 3.5. Схема данных и API

### Основные модели данных:
1.  **User:** `id, email, password_hash, full_name, avatar_url`.
2.  **Course:** `id, title, description, price, mentor_id, thumbnail_url`.

### Ключевые эндпоинты API:
* `POST /api/auth/login` — Аутентификация пользователя.
* `GET /api/courses` — Получение списка курсов.
* `GET /api/courses/:id` — Получение данных конкретного курса.
* `GET /api/user/me` — Получение данных текущего профиля.
